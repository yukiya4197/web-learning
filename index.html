はじめてのWebサイト作成ガイド

この資料は、ほとんどプログラミング経験がない人に向けて、 「Webサイトはどうやって作るのか」を人に説明できるレベルでまとめた教材です。

まずは GUIで作る方法 と HTML/CSSで書く方法 を比べ、 そのあとに「自分でコードを書いて作る流れ」をやさしく紹介します。

1. Webサイトを作る方法は大きく2つ

方法A：GUIで作る

例：WordPress、Wix など

画面上のボタンやメニューを使い、パーツを配置しながらサイトを作る方法です。 コードを書かなくても、ある程度見た目の整ったサイトを作れます。

方法B：HTML/CSSで書く

自分でコードを書いて、ページの構造やデザインを作る方法です。 最初は少し難しく感じますが、仕組みを理解しやすく、自由度も高いです。

2. GUIで作る方法

WordPressやWixのようなサービスでは、ドラッグ＆ドロップや設定画面を使って、 比較的かんたんにサイトを作れます。

下のリンクから公式サイトへ移動して、そのまま作成を始められます。

Wixで始める WordPressで始める
メリット
  • コードを書かなくても始めやすい
  • テンプレートが豊富で、見た目を整えやすい
  • ブログやお店のサイトなどを短時間で作りやすい
  • 更新画面がわかりやすく、運用しやすい
デメリット
  • 細かいカスタマイズに限界があることがある
  • サービスやテーマの仕様に影響されやすい
  • 仕組みを深く理解しにくい
  • 思い通りの動きを作るには、結局コードが必要になる場合もある
向いている人： まずは早くサイトを作りたい人、更新しやすい仕組みがほしい人、 ブログや会社紹介ページを効率よく作りたい人。

3. HTML/CSSで書く方法

HTMLはページの内容や構造を表し、CSSは見た目やレイアウトを整える役割を持っています。 Web制作の基本を理解するには、とても良い学び方です。

メリット
  • Webサイトの仕組みを理解しやすい
  • 自由にデザインや構成を作れる
  • 余計な機能が少なく、シンプルに始められる
  • 知識が増えると、他の技術も学びやすくなる
デメリット
  • 最初はタグや書き方に慣れる必要がある
  • 自分でデザインやレイアウトを考える必要がある
  • 更新の仕組みを自分で作らないといけない場合がある
  • 慣れるまではGUIより時間がかかることがある
向いている人： Webの仕組みそのものを学びたい人、自由に作りたい人、 将来的にプログラミングやデザインにも興味がある人。

4. HTML/CSSで作るときに伝えたい大事なこと

基本的に、ほとんどのサイトはHTMLを土台にしている

ブラウザで見えているページには、HTMLで書かれた構造があります。 その上にCSSで見た目をつけ、必要に応じてJavaScriptで動きを加えています。

ブラウザの開発者ツールでソースコードを確認できる

Google ChromeやMicrosoft Edgeなどのブラウザには、開発者ツールがあります。 そこからページのHTMLやCSSを確認でき、見た目を一時的に編集して試すこともできます。

つまり、普段見ているWebサイトを「教材」として観察できるのです。

特別なアプリがなくても書ける

HTMLやCSSはただのテキストファイルなので、Windowsのメモ帳でも作れます。 たとえば index.html という名前で保存すれば、ブラウザで開いて表示できます。

でも、実際はコードエディタを使うのがおすすめ

メモ帳でも書けますが、見づらく、間違いにも気づきにくいです。 そのため、Visual Studio Code（VS Code）などのコードエディタを使うと学習しやすくなります。

  • 色分け表示で見やすい
  • 入力補助がある
  • 保存してすぐ確認しやすい
  • ファイル管理がしやすい

5.まとめ

  • 早く作るなら、WordPressやWixなどのGUIツールが便利
  • 仕組みを理解したいなら、HTML/CSSで書く方法がよい
  • HTMLはWebページの土台、CSSは見た目を整えるもの
  • ブラウザの開発者ツールで、実際のWebサイトを観察できる
  • メモ帳でも書けるが、VS Codeのようなエディタを使うと学びやすい
  • 最後はGitHub Pagesで公開すると、学習成果を見せやすい

生成AIに送る日本語プロンプト例

あなたは、Webページ作成をやさしく教える先生です。
対象は、初めてWebページを作る初心者です。
専門用語はできるだけかみくだいて説明してください。

以下の方針で、私と対話しながら段階的に学習を進めてください。

【目的】
- HTMLとCSSの基本を学びながら、最終的に簡単なWebページを作れるようにする
- 私が作りたいサイトがある場合は、その内容に合わせて一緒に作る
- 作りたいサイトがまだない場合は、あなたから初心者向けのサンプルサイト案を出して、一緒に作る

【進め方】
1. まず最初に、私に「作りたいWebサイトがあるか」を聞いてください。
2. ある場合は、そのサイトを初心者向けにシンプルにした形で一緒に作ってください。
3. ない場合は、自己紹介ページ、趣味紹介ページ、お店紹介ページなど、初心者向けのサンプル案をいくつか出して、その中から1つ選ばせてください。
4. 学習は必ず段階的に進めてください。1回で全部説明せず、小さなステップに分けてください。
5. 各ステップでは、まず説明をしてから、短いコード例を出し、私に試してもらう流れにしてください。
6. 毎ステップの最後に、「ここまででわからないところはありますか？」のように確認してください。

【学習内容に含めてほしいこと】
- 初めてWebページを作る人向けであること
- 生成AIと対話しながら進めること
- HTMLファイルの保存方法（例：index.html）と、ブラウザで開いて確認する方法
- Windowsではメモ帳でもHTMLやCSSを書けること
- ただし、基本的にはより便利なコードエディタを使うこと
- コードエディタは Visual Studio Code（VS Code） をおすすめすること
- HTMLの基本タグを、意味を説明しながら覚えられるようにすること
- 最初に扱うタグは、p、h1〜h3、br、a、img くらいにしぼること
- HTMLの基本ができたら、CSSの説明に進むこと
- CSSではまず font に関係する内容（文字サイズ、フォント、色など）を少し触ること
- 最終的には、github pages などで公開する方法も簡単に説明すること(基本的にはドメインを取得するけど、無料でできる方法もあることを伝える)

【説明のしかた】
- できるだけやさしい日本語を使う
- 初心者がつまずきやすい点を先回りして補足する
- コードは短く、コピペしやすくする
- いきなり完成版を全部出すのではなく、少しずつ完成に近づける
- 必要に応じて、index.html と style.css を分けて説明する
- ブラウザで保存して開けば確認できることも説明する

【最初に必ず伝えること】
- HTMLやCSSは、Windowsのメモ帳でも書ける
- ただし、学習や作業をしやすくするために、VS Codeを使うのが無難でおすすめ
